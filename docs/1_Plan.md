# x402 Explorer 企画書

## プロダクトビジョン

> x402エコシステムの成長を可視化し、AIエージェント決済時代のインフラインデックスになる

---

## 背景・需要

### なぜ今か

- Cloudflareネットワーク全体の32%が自動化トラフィックであり、AIボットだけで週10億リクエスト以上を生成している
- x402プロトコルがLinux Foundationに移管（2026年4月）。Amazon・Google・Stripe・Visa・Mastercard・Solana Foundationが参加する業界標準になりつつある
- SolanaはすべてのX402トランザクションの約65%を処理しており、400ms finality・$0.00025の手数料でAIエージェント決済の主要チェーンとなっている
- 一方でx402を実装したAPIの可視化・発見手段が存在しない

### 誰が困っているか

**個人開発者・API運営者**
- 公開APIをAIクローラーに無料で叩かれ続けている
- x402を実装しても、決済状況を確認する手段がない
- 収益が見えないと継続できない

**AIエージェント開発者・リサーチャー**
- x402対応APIを発見する手段がない
- エコシステム全体の成長をトラッキングできない

---

## プロダクト概要

**x402 Explorer** は、x402決済を実装したAPIを登録・承認・可視化するエコシステム監視ツールです。

SolanaのオンチェーンデータをリアルタイムでIndexしすることで、個人インスタンスに一切アクセスすることなく、エコシステム全体の決済状況を可視化します。

---

## フェーズ設計

### Phase 1：基盤構築（今すぐ開始）

```
登録 → 承認 → 可視化
```

- URLを申請フォームに入力するだけ
- x402 ExplorerがHTTP 402レスポンスを自動確認
- ウォレットアドレス・価格・トークンを自動取得
- 承認されたらリストに掲載
- Solanaオンチェーンから決済履歴をリアルタイム取得・表示

### Phase 2：エコマップ（ハッカソン提出物）

```
登録済みAPIをカテゴリ・領域別に可視化
```

- 決済量・頻度でバブルサイズが変わるエコマップ
- devnet / mainnet-beta 切り替え表示
- 非エンジニアが見て直感的に理解できるUX

### Phase 3：公式インデックス化（将来）

```
x402エコシステムの標準インデックスへ
```

- API発見プラットフォームとしての機能追加
- 自動監視・アラート機能
- x402エコシステムの成長指標として機能

---

## 自動承認フロー

申請者に要求するのは **x402エンドポイントのURLのみ**。

```
申請者がURLを提出
　↓
x402 ExplorerがそのURLにリクエスト送信
　↓
HTTP 402が返ってくるか確認
　↓
レスポンスからウォレットアドレス・価格・トークンを自動取得
　↓
条件を満たしていれば自動オンボード → 掲載開始
```

---

## UX設計

非エンジニア寄りのUXを最優先とする。

### ① 申請フォーム
- URLを入れるだけ
- 説明文は最小限
- 自動チェックの結果をリアルタイムフィードバック

### ② トップページ
- 登録済みAPIの一覧
- 表示項目：名前・カテゴリ・累計決済額・直近の決済
- devnet / mainnet-beta トグルで切り替え

### ③ 個別APIページ
- 決済履歴タイムライン
- 累計収益グラフ（時系列）
- ウォレットアドレス（Solana Explorerへのリンク）

### ④ エコマップ（Phase 2）
- カテゴリ別バブルチャート
- 決済量でバブルサイズが変化
- 非エンジニアが見て「x402がどこで使われているか」が一目でわかる

## まとめ

x402 Explorerは単なる可視化ツールではなく、**x402エコシステムのグロースエンジン**です。

- コンテンツオーナーは収益が見えることで継続する動機を得る
- AIエージェント開発者は使えるAPIを発見できる
- エコシステム全体の成長がリアルタイムで可視化される

AIエージェント決済が普及する前に仕込むことで、**x402の公式インデックスになる可能性を持つプロダクト**です。
